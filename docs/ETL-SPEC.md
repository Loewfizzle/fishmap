## Core Contracts

- All output in EPSG:4326.
- Every access point **MUST** have ≥1 entry in `sources[]` with `name`, `url`, `retrieved`.
- `last_verified` is a manual or ETL audit date (ISO).
- `manifest.json` is the provenance record (etl_run_date, per-source sha256 + record counts + coverage_notes).
- `make etl-sample` (or `python -m scripts.etl_sample`) must always succeed in clean clone + produce valid outputs that pass `make etl-validate`.

## AOI

Exact 40-mile buffer (see `scripts/aoi.py` and committed `data/aoi.geojson`).

Center: `-85.6681, 42.9634` (Grand Rapids).
Radius: 40 statute miles → 64373.76 m.
Projection for buffer: EPSG:3857 (Web Mercator) as approximation (documented in aoi.py).

## Classification (PR 1 minimal implementation)

Implemented in `scripts/classify.py:get_sample_sites()` + `classify_access_point()`:

- For the 4 hand-curated real sites, directly yields:
  - Fish Ladder Park → `access_type: "bank"`, `access_quality: "high"`, `inferred: false`
  - Johnson Park Shoreline → `bank`, `high`
  - Richmond Park Fishing Pier → `pier`, `high`
  - Reeds Lake → `bank`, `high`
- Matches DESIGN pseudocode "1. Direct DNR attrs (highest confidence)" path for known public shore/pier locations.
- Full spatial (30m hydro buffer + public parks intersect - private parcels + road-end) deferred to PR 6 with real gdfs.

## Enrichment

`scripts/enrich.py` attaches parking, facilities, ada, species[], regulations (with DNR URL), notes, and the complete `sources[]` + `last_verified`.

## Export / Validation

`etl_sample.py` builds the FeatureCollection and runs inline validation (equivalent rules in `validate_sample.py`):

- Required: id, name, waterbody, access_type, access_quality, lat, lon, sources, last_verified
- Enums: access_type ∈ {bank,dock,pier,wade,road_end,park_shore}
- Every feature must have non-empty sources[] with full citation triples
- Geometry: valid Point, coordinates in rough AOI bbox
- manifest must record etl_run_date + source provenance + output sha256

## Draft JSON Schema (access point)

Full machine-readable schema will live at `docs/access-point-schema.json` (future PR). This is the PR1 draft used for validation + docs.

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "Fishmap Access Point (PR 1 draft)",
  "type": "object",
  "required": ["id", "name", "waterbody", "access_type", "access_quality", "lat", "lon", "sources", "last_verified"],
  "properties": {
    "id": { "type": "string" },
    "name": { "type": "string" },
    "waterbody": { "type": "string" },
    "access_type": {
      "type": "string",
      "enum": ["bank", "dock", "pier", "wade", "road_end", "park_shore"]
    },
    "access_quality": {
      "type": "string",
      "enum": ["high", "medium-high", "medium", "low"]
    },
    "lat": { "type": "number", "minimum": -90, "maximum": 90 },
    "lon": { "type": "number", "minimum": -180, "maximum": 180 },
    "sources": {
      "type": "array",
      "minItems": 1,
      "items": {
        "type": "object",
        "required": ["name", "url", "retrieved"],
        "properties": {
          "name": { "type": "string" },
          "url": { "type": "string", "format": "uri" },
          "retrieved": { "type": "string", "pattern": "^\\d{4}-\\d{2}-\\d{2}$" }
        }
      }
    },
    "last_verified": { "type": "string", "pattern": "^\\d{4}-\\d{2}-\\d{2}$" },
    "parking": { "type": "object" },
    "facilities": { "type": "array", "items": { "type": "string" } },
    "hours": { "type": "string" },
    "ada": { "type": "string" },
    "species": { "type": "array", "items": { "type": "string" } },
    "regulations": {
      "type": "object",
      "properties": {
        "summary": { "type": "string" },
        "url": { "type": "string", "format": "uri" }
      }
    },
    "notes": { "type": "string" },
    "inferred": { "type": "boolean" },
    "needs_review": { "type": "boolean" }
  }
}
```
